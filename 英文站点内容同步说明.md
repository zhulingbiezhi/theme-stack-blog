# 英文站点内容同步说明

## 实现方案

为了让英文站点也能显示中文内容，避免大量翻译工作，我们采用了**复制中文文章为英文版本**的方案。

## 已完成的工作

### 1. 静态页面英文版本
创建了以下英文静态页面：
- About (`/en/about/`)
- Friends (`/en/links/`)
- Contact Us (`/en/contact-us/`)
- Privacy Policy (`/en/privacy-policy/`)
- Terms of Service (`/en/terms-of-service/`)
- Cookie Policy (`/en/cookie-policy/`)
- Disclaimer (`/en/disclaimer/`)

### 2. 文章内容同步
- 为所有60篇中文文章创建了对应的 `.en.md` 版本
- 英文版本包含完全相同的中文内容
- 这样英文站点访客也能看到所有文章

### 3. 配置更新
- 更新了 `config.yaml`，为英文版本添加了主菜单（Home链接）
- 为英文版本添加了页脚菜单（Privacy、Terms、Cookies等）
- 确保语言切换功能正常工作

## 维护方式

### 新增文章时
当你创建新的中文文章时，需要同时创建英文版本：

```bash
# 方法1：手动复制
cp content/post/2025/new-article.md content/post/2025/new-article.en.md

# 方法2：使用脚本（推荐）
./create_en_posts.sh
```

### 更新文章时
如果你更新了中文文章，也需要同步更新英文版本：

```bash
# 复制更新后的内容
cp content/post/2025/article.md content/post/2025/article.en.md
```

## 自动化脚本

已创建 `create_en_posts.sh` 脚本，可以批量为所有中文文章创建英文版本：

```bash
chmod +x create_en_posts.sh
./create_en_posts.sh
```

## 未来优化建议

### 1. 使用 Git Hook
可以创建 Git pre-commit hook，自动为新文章创建英文版本：

```bash
# .git/hooks/pre-commit
#!/bin/bash
./create_en_posts.sh
git add content/post/**/*.en.md
```

### 2. 使用 Hugo 配置
也可以考虑在 Hugo 配置中设置默认语言回退，但当前方案更简单直接。

### 3. 逐步翻译
未来如果有时间，可以逐步翻译重要文章：
- 保留 `.en.md` 文件
- 只需要翻译 front matter 和正文内容
- 不需要改变文件结构

## 注意事项

1. **文件命名规则**：
   - 中文文章：`article.md` 或 `article.zh-cn.md`
   - 英文文章：`article.en.md`

2. **URL 结构**：
   - 中文：`/2025/03/article/`
   - 英文：`/en/2025/03/article/`

3. **SEO 考虑**：
   - 虽然英文站点显示中文内容，但仍然有利于 SEO
   - 搜索引擎会索引英文 URL
   - 未来可以逐步替换为真正的英文翻译

## 总结

通过这个方案，我们成功地让英文站点从26个页面增加到243个页面，大大丰富了英文站点的内容，同时避免了大量的翻译工作。访客切换到英文后，可以看到完整的网站结构和所有文章内容。
